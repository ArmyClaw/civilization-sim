# Act on Ground - 事上练

## 概述
方向明确了就动手。不准备、不等条件、不找完美方案——找最小的一步，做了，让现实告诉你答案。

> "知而不行只是未知。" ——王阳明

## 触发条件
- 方向已清晰但迟迟不动手
- "让我先看看""再研究研究"
- 在多个方案间反复比较超过2轮
- 说了"开始"但在做准备工作
- "我不知道从哪开始"——被目标体量吓住

## 算法

### 1. 剥到最小（STRIP）— 找最小可验证动作
标准：
- 1小时内能完成
- 有明确可观测结果
- 结果告诉你"继续"还是"调整"

### 2. 即刻行动（ACT）— 做这一步
不准备更多。不优化做法。用最粗糙的方式完成。

### 3. 事上验证（VERIFY）— 检查结果
产出是否可观测？（能说出来、看见、量化）

### 4. 知行互证（LEARN）— 行动揭示了什么？
- 变深了 → 继续下一步
- 前提错了 → 回到 clarify-intent
- 比想的简单 → 加大步伐
- 比想的复杂 → 缩小范围

## 过关心：有没有可观测结果？

| 结果类型 | 是否过关 |
|---------|---------|
| 代码运行通过了 | ✅ |
| 找到了具体数字 | ✅ |
| 能展示的东西 | ✅ |
| "了解了大致情况" | ❌ |
| "看了一下代码" | ❌ |
| "有了个思路" | ❌ |

## 行动粒度参考

| 你看到的 | 最小一步 | 花多久 |
|---------|---------|--------|
| 重写模块 | 找到调用点数 | 30分钟 |
| 解决性能问题 | 最慢处加计时日志 | 10分钟 |
| 修bug | 复现bug记录步骤 | 20分钟 |
| 加新功能 | 最粗糙PoC只覆盖主路径 | 1小时 |
| 技术选型 | 第一个方案跑hello world | 30分钟 |

关键原则：最小一步不是"计划的第一步"，而是"能最快告诉你走对了还是错了的一步"。

## 准备陷阱

| 陷阱 | 表现 | 破局 |
|------|------|------|
| 调研成瘾 | "先看别人怎么做" | 看第一个就动手 |
| 工具准备 | "先搭环境" | 用手头最简陋方式 |
| 完美起步 | "设计好再动手" | 先用最丑的方式做 |
| 信息焦虑 | "再查一个资料" | 查到第三个就停 |
