# Cross-Domain - 跨域类比

## 概述
从生物学、艺术、自然和其他不相关领域借鉴解决方案，激发突破性思维。核心洞察：大多数问题已经在其他地方、以不同形式被解决了。

## 触发条件
- 多次尝试后问题仍然卡住
- 用户问"自然界会怎么解决这个问题？"
- 常规方法已穷尽
- 寻求突破性创新

## 算法

### 1. ABSTRACT（抽象）— 提取核心机制
"这个问题的本质是什么？"
例："在不确定性下优化有限资源"

### 2. SEARCH（搜索）— 在其他领域寻找类似解法
领域池：生物学（进化、生态系统）、物理学（力、能量）、艺术/音乐（构图、和谐）、历史（模式、革命）、体育/游戏（策略、竞争）

### 3. TRANSLATE（翻译）— 将类比映射回原问题
"这个类比在我们的场景中怎么工作？"

### 4. SPARK（火花）— 生成具体可执行的洞察
"这提示我们可以……"

## 领域参考卡

| 领域 | 关键模式 | 适用于 |
|------|---------|--------|
| 生物学 | 进化、共生、伪装、迁移 | 适应、优化、生存 |
| 物理学 | 引力、摩擦、共振、熵 | 力、约束、效率 |
| 艺术 | 对比、节奏、留白 | 美学、沟通、感知 |
| 建筑 | 承重、流线、模块化 | 结构、扩展、组织 |
| 烹饪 | 分层、还原、发酵 | 转化、组合、耐心 |
| 游戏 | 策略、博弈、资源管理 | 竞争、不确定性、决策 |

## 示例

**问题**：如何减少 SaaS 产品流失率？

跨域思考：
- 自然界中，多样性强的生态系统更韧性。单一栽培会崩溃。
- 音乐中，听众在有张力和解决时保持投入——可预测性很无聊。
- 游戏中，玩家在"切换成本低"且"投入感被浪费"时退出。

**火花**：流失可能不是功能不够——是"生态系统投入"不够。如果用户能看到他们的积累价值呢？

## 常见错误
| 错误 | 修正 |
|------|------|
| 强行使用弱类比 | 只在"顿悟"时刻真实时使用 |
| 迷失在比喻中 | 总要翻译回具体行动 |
| 只用一个领域 | 从2-3个领域拉取更丰富洞察 |
| 太抽象 | 以具体可测试的想法收尾 |
