# Reflect Anchor - 知行复盘

## 概述
做了一段时间，停下来照镜子。方向偏了没有？行动验证了什么？推翻了什么？"我以为对但其实错的是什么？"——答不上来这道题，复盘就是走过场。

> "知之真切笃实处即是行，行之明觉精察处即是知。" ——王阳明

## 触发条件
- 完成了一个阶段（3-5步之后）
- 一步失败了
- "感觉不对""是不是跑偏了"
- 连续做了很多但没看过目标
- "做完了"但没对照目标检查

## 算法

### 1. 锚定初心（ANCHOR）
回到最开始的清晰方向。原话写出来，不改。现在做的事和目标什么关系？

### 2. 事实验证（EXAMINE）
列出每一步和实际结果。哪些验证了假设？哪些推翻了假设？只看事实。

### 3. 直面未知（CONFRONT）— 核心
"我以为对但其实错的"是什么？
不是"哪里做错了"（追责），是"我带着什么错误假设在行动"（认知更新）。

### 4. 校准方向（CALIBRATE）
目标还是原来的吗？
- 是 → 继续，调整路径
- 不是 → 回到 clarify-intent

## 过关心：能否说出"我以为对但其实错的"？

| 状态 | 含义 |
|------|------|
| 能具体说出来 | 复盘有效，认知已更新 |
| 太笼统 | 再挖一层 |
| 说不出来 | 要么复盘太浅，要么行动太安全 |

## 复盘深度

| 层级 | 问什么 | 产出 |
|------|-------|------|
| L1: 事实层 | 做了什么？结果？ | 时间线+结果 |
| L2: 认知层 | 哪些假设被推翻？ | "我以为X，其实是Y" |
| L3: 模式层 | 错误假设怎么形成的？ | "我倾向于低估XX" |

必须达到L2才算过关。

## 示例

| 步骤 | 预期 | 实际 | 结论 |
|------|------|------|------|
| grep auth引用 | ~7个文件 | 17个文件 | 影响范围被低估 |
| 分析间接依赖 | 少量 | 11个间接依赖 | 耦合更深 |

**我以为对但其实错的**：以为是重构问题，其实是架构问题。
**校准**：不能直接重构，需要先建抽象层。

## 常见错误

| 错误 | 修正 |
|------|------|
| 变成自我批评 | 不是追责，是校准 |
| 全是好消息 | 3步没发现错误假设=复盘太浅 |
| 回到白板重新规划 | 复盘完应该有具体下一步 |
| 跳过"直面未知" | 这是最有价值的部分 |
