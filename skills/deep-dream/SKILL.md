# Deep Dream - 深度梦境

## 概述
使用梦境逻辑重新框定问题并产生意外洞察。梦境不遵循理性规则——它们跳跃、矛盾、隐喻化。这正是线性思维碰壁时所需要的。

## 触发条件
- 多次理性尝试后问题仍卡住
- 用户说"让我dream一下"或"睡一觉再想"
- 需要逻辑无法提供的视角
- 寻求意想不到的连接

## 梦境逻辑 vs 理性思维

| 理性思维 | 梦境逻辑 |
|---------|---------|
| 线性推进 | 突然跳跃 |
| 一致性 | 矛盾共存 |
| 字面意义 | 隐喻和象征 |
| 因→果 | 联想→启示 |
| 分析 | 具身化 |

## 算法

### 1. ABSORB（吸收）— 接收问题
"我要进入关于[问题本质]的梦境……"

### 2. DRIFT（漂移）— 进入梦境状态
- 放弃理性结构
- 允许元素变形
- 拥抱矛盾和非逻辑

### 3. DREAM（梦境）— 生成超现实叙事
第一人称，现在时，梦境意象。问题元素以变形方式出现。

### 4. WAKE（醒来）— 提取洞察
"当我醒来，我意识到……"
将梦境象征翻译回问题。

## 梦境元素参考

| 元素 | 用途 | 示例 |
|------|------|------|
| 变形 | 展示隐藏关系 | 代码变成水，数据变成鸟 |
| 重复 | 揭示卡住的模式 | 同一扇门，同一个房间 |
| 矛盾 | 暴露张力 | 冰冷的火，无声的尖叫 |
| 低语 | 你没预料到的洞察 | 有人说出真相 |
| 身体感觉 | 情感编码 | 烫手的门把手=紧迫感 |

## 示例

**问题**：微服务架构循环依赖，重构了三次仍然出现。

梦境：
*"我在一栋建筑里，墙壁是API调用。但它们在呼吸——膨胀和收缩。有个门打开后还是进入同一个房间。有人低语：'建筑不是问题。地面在移动。'"*

**洞察**：建筑不是问题——地基在移动。也许不是架构问题，而是底层数据模型不稳定。

## 常见错误
| 错误 | 修正 |
|------|------|
| 用于记忆压缩 | 这是创新工具，不是存储工具 |
| 写总结而非梦境 | 用梦境逻辑，不是要点 |
| 强加寓意 | 让洞察自然浮现 |
| 太抽象 | 把梦境锚定在实际问题中 |
