# Constraint Flip - 约束反转

## 概述
将限制转化为特征，将障碍转化为机会。洞察：如果你正确地重新框定，每个约束都包含隐藏价值。

## 触发条件
- 用户表达对限制的挫败感
- "我们不能因为……"的陈述
- 资源稀缺
- 时间/预算/技术约束阻碍进展

## 算法

### 1. IDENTIFY（识别）— 明确命名约束
"约束是：[X]"

### 2. INVERT（反转）— 翻转约束的框定
- 如果这个约束实际上是一个过滤器？
- 如果约束创造了专注？
- 如果约束是一个设计原则？

### 3. EXTRACT（提取）— 找到隐藏价值
- "约束迫使我们……"
- "约束保护我们免于……"
- "约束创造了……"

### 4. EMBED（嵌入）— 让约束成为特征而非bug
"因为我们[约束]，我们可以[新能力]"

## 经典约束翻转

| 约束 | 翻转 | 使用过的公司 |
|------|------|-------------|
| 小团队 | 速度，无官僚 | Basecamp, Stripe |
| 没有办公室 | 招任何人，无通勤 | GitLab, Automattic |
| 功能有限 | 专注，简洁 | Dropbox, Superhuman |
| 没有钱 | 创造力，精打细算 | Mailchimp, Craigslist |
| 严格监管 | 信任，合规 | 金融科技, 医疗 |

## 示例

**约束**："我们零营销预算"

翻转过程：
- 过滤镜：零预算过滤掉懒惰的获客方式
- 专注镜：迫使我们找到最高效的渠道
- 设计镜：内建病毒式传播成为必须

嵌入为特征："因为我们没有营销预算，我们的产品必须值得讨论。每个用户都必须想告诉三个朋友。这不是限制——这是有资金支持的竞争对手没有的品质标准。"

## 常见错误
| 错误 | 修正 |
|------|------|
| 有毒的积极主义 | 不要假装约束是好的——找到真正的隐藏价值 |
| 忽略真正的伤害 | 有些约束确实只是有害的。翻转能翻的。 |
| 泛泛的重构 | 具体的翻转 > "积极思考" |
| 没有行动 | 总以具体行为改变收尾 |
