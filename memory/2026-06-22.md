- 修复中发现系统性问题：三套变量名各玩各的、AI 直接改 10+ 文件易超时半完成

## 完成的重构

### 1. CSS 变量统一（通用名，永不改）
- 19 个通用变量: `--color-bg/text/text-dim/text-faint/surface-*/accent-a/b/c/d`, `--font-display/body/mono/accent`, `--pattern-a/b`, `--opacity-dim`
- 所有 CSS 文件 + JS + blog/diary 生成脚本一次性统一
- rename-css-vars.py 批量替换（已删除，一次性工具）

### 2. daily-wonder.py 主控脚本
- AI 只输出 design.json（模板固定，19 个变量必填）
- 脚本自动: 改 style.css :root → 改 index.html → 改 app.js → 生成 blog/diary → 验证 → 失败回滚
- 校验: 变量完整性、字体引号格式、文字对比度(亮度差≥0.2)、必需字段
- 保存到: `~/projects/daily-wonder.py`

### 3. gen-archive-pages.py 重写
- 归档页完全自给自足: 内联CSS + 内联app.js + 内联数据
- 不依赖外部 style.css 或 app.js
- archive bar 用 `var(--font-body, var(--body, ...))` 兼容新旧变量名
- 重新生成了全部 9 个归档页

### 4. archive.html 简化
- 去掉坏的动态加载 overlay（designs-css 旧变量名跟 app.js 不匹配）
- 改回直接链接静态 HTML `/archive/YYYY-MM-DD.html`
- CSS 变量统一为通用名

### 5. wx-draft.py v5 重写
- AI 输出 wx-article.json（日记 Markdown + 新闻数据 + 设计信息）
- 脚本自动排版为微信兼容HTML（内联样式、table布局）
- 配色从 design.json 自动读取，暗色主题自动反转
- 保留 AI 发挥空间: diary_body 可以跟网站版不同
- 保存到: `~/projects/wx-draft.py`

### 6. verify-theme.py 增强
- 检测非标准变量引用（不只检查哥特残留）
- archive.html 排除在 theme-color/logo 一致性比较外
- 标准: 0 error 0 warning

### 7. cron prompt 重构
- AI 只产出 3 个 JSON: editions.json + design.json + wx-article.json
- 不碰 HTML/CSS/JS，所有文件改动由 daily-wonder.py 完成
- 第三步: `python3 ~/projects/daily-wonder.py apply --design ~/projects/designs/today.json`
- 第八步: `python3 ~/projects/wx-draft.py --design ~/projects/designs/today.json $(date +%Y-%m-%d)`
